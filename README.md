# Ads Optimizer — แดชบอร์ดโฆษณา 3 แพลตฟอร์ม

เว็บแอปสำหรับเช็คยอดโฆษณาและรับคำแนะนำการ optimize รายเดือน รวม **Facebook Ads, Google Ads, TikTok Ads** ไว้ในที่เดียว ทำงานทั้งหมดในเบราว์เซอร์ (ไม่ต้องมี server, ไม่มีฐานข้อมูล) — อัปโหลดไฟล์ CSV ที่ export จากแต่ละแพลตฟอร์ม แล้วระบบจะรวมยอด คำนวณ metric สำคัญ และแนะนำสิ่งที่ควรทำเดือนหน้าให้อัตโนมัติ

## ทำอะไรได้บ้าง

- **รวมยอด 3 แพลตฟอร์ม** — งบที่ใช้, impressions, clicks, CTR, CPC, conversions, CPA, ROAS
- **รองรับไฟล์ .csv และ .xlsx** — อัปโหลดได้ทั้งไฟล์ CSV และ Excel โดยตรง ไม่ต้องแปลงก่อน
- **แยกตามหน่วยธุรกิจ Yushi** — Bigfan, Industrial, Supply, Group, F&B (Shinsen), Denki, Marketing Technology (YMT), System, Solution ตัดข้อมูลข้ามทุกแพลตฟอร์ม มีทั้งฟิลเตอร์ด้านบน (ใช้กับทุกหน้า) และหน้า "หน่วยธุรกิจ" ที่สรุปแยกแต่ละบริษัทเป็นหัวข้อ
- **เลือกดูตามเดือน + บันทึกถาวร** — เก็บข้อมูลแยกแต่ละเดือนและบันทึกไว้ในเบราว์เซอร์อัตโนมัติ (คงอยู่แม้ปิด/รีเฟรชเว็บ) เปลี่ยนเดือนที่แถบซ้าย แดชบอร์ดทุกหน้าจะอัปเดตตามเดือนนั้นทันที จัดการ/ลบข้อมูลแต่ละเดือนได้ที่หน้า "จัดการข้อมูล"
- **เปรียบเทียบเดือน** — หน้า "เปรียบเทียบเดือน" แสดงตารางและกราฟแนวโน้มทุกเดือนที่เก็บไว้ พร้อมไฮไลต์เดือนที่ดีที่สุดและเดือนที่ควรปรับ (ดูตามหน่วยธุรกิจที่กรองไว้ได้ด้วย)
- **กราฟเปรียบเทียบ** สัดส่วนงบและประสิทธิภาพรายแพลตฟอร์ม + ตารางรายแคมเปญ (มีคอลัมน์หน่วยธุรกิจ)
- **คำแนะนำ optimize อัตโนมัติ (rule-based)** — จัดอันดับตามผลกระทบ เช่น แคมเปญที่ใช้เงินแต่ไม่มีผลลัพธ์, CPA สูงเกินเป้า, ROAS ต่ำ, CTR ต่ำ, และโอกาสเพิ่มงบในแคมเปญที่ทำได้ดี (ระบุหน่วยธุรกิจในแต่ละคำแนะนำ)
- **AI insights (ไม่บังคับ)** — ใส่ API key ของคุณเอง (Claude หรือ GPT) เพื่อรับบทวิเคราะห์เชิงกลยุทธ์เพิ่มเติม
- **ตั้งเป้าหมายเองได้** — CPA / ROAS / CTR ที่รับได้ เพื่อให้ระบบรู้ว่าแคมเปญไหน "ดี" หรือ "ควรปรับ"
- **พิมพ์เป็น PDF** สำหรับรายงานรายเดือน

## หน่วยธุรกิจ Yushi — วิธีให้ระบบจับได้

ระบบแยกหน่วยธุรกิจได้ 2 ทาง (เรียงตามความสำคัญ):

1. **จากชื่อไฟล์** — ถ้าตั้งชื่อไฟล์ให้มีชื่อหน่วยธุรกิจ เช่น `Yushi Industrial.csv` ระบบจะจัด **ทุกแคมเปญในไฟล์นั้น** เข้าหน่วยเดียวกันทั้งหมด เหมาะกับ **Google Ads** ที่ชื่อแคมเปญในไฟล์มักไม่มีชื่อบริษัท
2. **จากชื่อแคมเปญ** — ถ้าชื่อไฟล์ไม่มีหน่วยธุรกิจ ระบบจะดูจากชื่อแต่ละแคมเปญ เช่น `Messages-Yushi Industrial (Exhaust) YMT` เหมาะกับ Facebook/TikTok ที่ชื่อแคมเปญมีชื่อบริษัทอยู่แล้ว

รองรับ: Bigfan, Industrial, Supply, Group, F&B / Shinsen, Denki, YMT / Marketing Technology, System, Solution (ไม่สนตัวพิมพ์ และมีคำไทยบางส่วน) แคมเปญที่ไม่ตรงจะถูกจัดเป็น "ไม่ระบุหน่วยธุรกิจ" — แก้ keyword ได้ในไฟล์ `js/units.js` หรือใช้ `unit-checker.html` เพื่อทดสอบ

กฎพิเศษ: ชื่อที่ลงท้ายด้วย "Group" → เข้า Yushi Group, ชื่อที่มี "Yushi Industrial" + ลงท้าย YMT → เข้า Yushi Industrial

## การเลือกเดือน

ข้อมูลถูกผูกกับ "เดือนที่แสดงผล" ตอนอัปโหลด เลือกเดือนที่แถบซ้ายก่อนอัปโหลด CSV ของเดือนนั้น จากนั้นสลับเดือนเพื่อดูย้อนหลังได้ (แถบซ้ายจะบอกว่ามีข้อมูลเดือนไหนบ้าง) ทุกหน้าจะแสดงเฉพาะข้อมูลของเดือนที่เลือก

## วิธีใช้งานเบื้องต้น

แอปนี้ต้องเปิดผ่าน web server (ไม่ใช่ดับเบิลคลิกไฟล์ตรงๆ เพราะการโหลดข้อมูลตัวอย่างใช้ `fetch`)

```bash
# จากในโฟลเดอร์ ads-dashboard
python3 -m http.server 8000
# แล้วเปิด http://localhost:8000
```

จากนั้นกด **"โหลดข้อมูลตัวอย่าง"** ที่แถบซ้ายล่างเพื่อดูตัวอย่างทันที หรือไปที่หน้า **"จัดการข้อมูล"** เพื่ออัปโหลด CSV ของจริง

## วิธี export CSV จากแต่ละแพลตฟอร์ม

**Facebook Ads (Ads Manager)**
1. เปิด Ads Manager → แท็บ Campaigns
2. เลือกช่วงวันที่ (เช่น เดือนที่ผ่านมา) และคอลัมน์ที่ต้องการ (อย่างน้อย: Amount spent, Impressions, Link clicks, Results / Purchases, Purchase value)
3. กดปุ่ม **Reports → Export → Export table data (.csv)**

**Google Ads**
1. เปิดหน้า Campaigns (หรือ Keywords)
2. เลือกช่วงวันที่ และให้แน่ใจว่ามีคอลัมน์ Cost, Impr., Clicks, Conversions, Conv. value
3. กดไอคอน **Download (⬇️) → .csv**

**TikTok Ads Manager**
1. เปิด Campaign list เลือกช่วงวันที่
2. ตรวจว่ามีคอลัมน์ Cost, Impressions, Clicks, Conversions, Total complete payment value / ROAS
3. กด **Export → CSV**

> ระบบ auto-detect ชื่อคอลัมน์ได้ทั้งภาษาอังกฤษและภาษาไทย และชื่อคอลัมน์ที่ต่างกันเล็กน้อย หากบางคอลัมน์ตรวจไม่เจอ metric ที่เกี่ยวข้องจะถูกข้ามไป (ไม่ทำให้แอปพัง)

## วิธี deploy ออนไลน์ (ฟรี)

แอปเป็น static site ทั้งหมด เลือกวิธีใดก็ได้:

- **Netlify** — ลากโฟลเดอร์ `ads-dashboard` ทั้งโฟลเดอร์ไปวางที่ app.netlify.com/drop
- **Vercel** — `vercel` ในโฟลเดอร์ หรือเชื่อม GitHub repo
- **GitHub Pages** — push ขึ้น repo แล้วเปิด Pages ที่ branch หลัก
- **Cloudflare Pages** — เชื่อม repo, build command ว่างไว้, output directory = `/`

## เรื่องความปลอดภัยของ API key

API key สำหรับ AI ถูกเก็บไว้ใน `localStorage` ของเบราว์เซอร์คุณเท่านั้น และถูกส่งตรงไปยังผู้ให้บริการ AI (Anthropic/OpenAI) โดยไม่ผ่านเซิร์ฟเวอร์อื่น หากใช้ในเครื่องสาธารณะ แนะนำให้ลบคีย์หลังใช้งาน หากต้องการความปลอดภัยสูงสุดในระดับองค์กร ควรย้ายการเรียก AI ไปไว้หลัง backend proxy ของคุณเอง

## ข้อจำกัดและขั้นถัดไป

ปัจจุบันเป็นแบบ **อัปโหลด CSV รายเดือน** หากต้องการ **real-time จริง** (ดึงยอดอัตโนมัติ) ต้องต่อ official API ของแต่ละเจ้า ซึ่งต้องมี:
- Facebook: Marketing API + access token + App review
- Google Ads: API developer token + OAuth
- TikTok: Marketing API + approved app

สิ่งเหล่านี้ต้องมี backend คอยเก็บ token และ refresh ตามรอบ เป็นเฟสถัดไปที่ต่อยอดจากโครงสร้างนี้ได้
